# Domino から学ぶ設計メモ（JpoProducer 向け）

**参照ローカル:** `/mnt/c/Users/user/OneDrive/Desktop/Domino/`  
（`Domino.exe`, `Manual/use.htm`, `IniFiles/user/*.ini`, `Module/`）  
**公式:** https://takabosoft.com/domino

Domino は **ピアノロール中心の MIDI 専用エディタ**（フル DAW ではない）。  
JpoProducer のポジション（爆速スケッチ + SMF）と近い。**丸コピーではなく「完成度の軸」を借りる。**

---

## Domino の画面モデル（3ペイン）

| ペイン | 役割 | Jpo への示唆 |
|--------|------|----------------|
| **ピアノロール** | 時間×音高でノート編集 | Tab3 の本丸。ツールモードを混ぜない |
| **イベントリスト** | ノート/CC を数値で編集（Gate に `480*4` 等） | 将来: 選択ノートの数値インスペクタ |
| **イベントグラフ** | Vel / CC / PB をグラフ編集 | 将来: Tab3 下段 Vel レーン（MVP 後） |
| **トラックセレクト** | カレントトラック切替 | 既にトラック概念あり。オニオンと併用 |

公式が強調する完成度ポイント:

- オニオンスキン（他トラック重ね表示）
- 移動中の音階プレビュー
- 無制限に近い Undo/Redo
- Tick / Gate の **別々の** クォンタイズ
- ペン / 選択 / 消しゴムの明確なモード
- ヘッダークリックで再生位置（＝貼付基準）
- Space 再生
- SMF format0/1 読み、format1 書き
- `EnableSameNote`（同一ノート扱いの方針）

## ユーザー設定から読める方針（この PC の Domino）

`IniFiles/user/System.ini` より:

| キー | 値 | 解釈 |
|------|-----|------|
| `EnableSameNote` | **0** | 同一音の重ねを基本オフ（モノ的・被り抑制寄り） |
| `SelNotePrevBeat` | 32 | 選択時プレビュー関連 |
| `LoopSel` | 1 | 選択範囲ループしやすい |
| `PausePlayMode` | 1 | 再生停止の挙動 |

`PianoRoll.ini`:

| キー | 値 | 解釈 |
|------|-----|------|
| `OnionNoteGrad` | 1 | オニオン表示を重視 |
| `PianoGuide` | 1 | 鍵盤ガイド ON |
| `NoteGrad` | 1 | ノートのグラデ表示 |

→ Jpo の「オニオン（スケール/コード）」は Domino 文化と整合。**生成結果の同一 pitch 時間重なりは、Domino 利用者の感覚では“汚い”**。

## 操作モデル（Tab3 が揃えるべき MVP）

Domino マニュアル（`Manual/use.htm`）から抽出した **最小完成セット**:

1. **ペン:** クリック配置（デフォルト Gate/Vel）、右端で Gate、本体ドラッグで移動  
2. **選択:** 矩形選択 → 移動 / Delete / コピー  
3. **消しゴム or ダブルクリック削除**  
4. **プレイヘッド:** タイムライン/ヘッダークリック  
5. **ペースト位置 = プレイヘッド**（Domino も「位置を決めてから貼る」）  
6. **Snap:** 位置用と長さ用を概念として分ける（実装は後でも、仕様コメントに残す）  
7. **プレビュー:** ドラッグ中・配置時に音が鳴る  
8. **Space 再生 / 停止**  
9. **Undo**（Domino の信頼性の核。Jpo は未整備なら計画に入れる）

Jpo 既に近いもの: Select/Draw/Erase、playhead paste、Space、オニオン。  
ギャップになりやすいもの: **安定 ID による選択表示**、**同一 pitch 重なり方針**、**Undo スタック**、**Vel グラフ**。

## Domino に寄せないもの（Jpo の勝ち筋）

Domino は汎用打ち込み。Jpo は:

| Jpo 固有 | Domino には無い/弱い |
|----------|----------------------|
| Ch1 コードブロックが進行の SoT | コードは手打ち or 別ツール |
| Tab2 パターン生成（Piano/Bass/Drum） | 手動が前提 |
| 4タブで編集モデルを分離 | 単一ピアノロール中心 |
| 爆速スケッチ・生成プレビュー | 精密 CC/音源マクロ |

**禁止:** Domino 全機能を移植しようとして monofile をさらに肥大化させる。  
**推奨:** Domino を **Tab3 UX と MIDI データの“きれいさ”の基準** に使い、生成は Jpo 独自。

---

## パターン MIDI 編集のワークフロー

HANDOVER 通り、生成テンプレは Domino で直すのが正しい:

1. `assets/patterns/*.mid` を Domino で開く  
2. Gate を短く / 帯域を整える / 重なりを消す  
3. SMF 書き出し → リポジトリへ  
4. `cargo test` で生成回帰

エージェントは **Domino.exe を自動操作しない**（GUI 自動化なし）。ユーザー手編集 or 数値 assert で固定。

## 実装に落とすチェックリスト（Domino 由来）

- [ ] ノートは UI 上 `(start, gate/dur, pitch, vel, ch)`。SMF 変換は境界のみ  
- [ ] 選択・ハイライトは **永続 ID** のみ（座標や index に依存しない）  
- [ ] 同一 ch・同一 pitch の時間重なり方針を明示（Domino: 基本避ける）  
- [ ] ツールモードは排他（ペン操作中に選択ショートカットが漏れない）  
- [ ] プレイヘッドが「挿入点」の単一の真実  
- [ ] プレビュー音は編集フィードバック用、再生エンジンと分離して考えてよい  
- [ ] Undo は「プロジェクト状態のスナップショット or コマンド」単位（後フェーズ可）
